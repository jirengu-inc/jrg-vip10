【a.com 页面的 ifream 引用 b.com 页面的资源】

- 前端

为输入框 input 添加事件处理程序，事件类型为 message

- 服务端

为 window 添加事件处理程序，事件类型为 message

- 原理

postMessage 是 HTML5 中新增方法，可实现跨域通信；
但 postMessage 并不是向服务器读写资源，只是向外发送消息而已（相当于我们使用手机发送短信）
相当于：A 页面向 B 页面发送了一条消息，B 页面会接受到该消息，如果 B 页面需要该消息，则监听 message；否则无需关心该消息

- 发展观念

现在，我们就解除了降域的限制，可以将 postMessage 应用到各个不同的域之间，实现跨域
并且，该方式比较安全，因为对方并不能直接操控我方资源，仅仅是发了一条消息，相当于指令，而最终操作权限仍在自己手中

- 注意：

虽然解决了降域的限制，但是：
postMessage 是 window 的一个方法，话句话说：它的弱点是只能向 window 窗口发送消息；
所以使用时至少要有一个 window 才行，postMessage 不可以向其他域名发送消息
